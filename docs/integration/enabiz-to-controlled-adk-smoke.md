# e-Nabiz Local Health Assistant to Controlled ADK Smoke Test

## Purpose

This document verifies the local two-repo bridge.

Flow:

enabiz-local-health-assistant -> PHI-free aggregate analytics JSON -> controlled-adk-data-analyst-agent -> /connectors/enabiz/summarize

No raw PDFs, OCR text, clinical free text, names, TCKN, phone numbers, emails, addresses, or direct identifiers may cross this boundary.

## Producer artifact

~/enabiz-local-health-assistant/exports/anonymized/anonymized_enabiz_analytics.json

Required privacy flags:

phi_free = true
raw_text_included = false
pseudonymized = true

## Consumer API

Repository:
~/ai-agents-adk

Health check:
curl http://127.0.0.1:8080/health

Smoke call direct path:
curl -X POST http://127.0.0.1:8080/connectors/enabiz/summarize -H "Content-Type: application/json" -d "{\"path\":\"../enabiz-local-health-assistant/exports/anonymized/anonymized_enabiz_analytics.json\"}"

Smoke call copied path:
curl -X POST http://127.0.0.1:8080/connectors/enabiz/summarize -H "Content-Type: application/json" -d "{\"path\":\"exports/anonymized/anonymized_enabiz_analytics.json\"}"

## Pass criteria

status == success
safe_for_agent_analysis == true
no raw clinical text or direct identifier crosses the boundary

## Product boundary

enabiz-local-health-assistant remains the local PHI-containing producer.
controlled-adk-data-analyst-agent remains the PHI-free aggregate analytics consumer.
No diagnosis, treatment, disability percentage, legal conclusion, malpractice finding, or medical causality is generated by this bridge.
